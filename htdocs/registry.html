An OAI Static Repository Gateway

Welcome! Our Gateway service makes OAI Static Repositories harvestable via the Open Archives Protocol for Metadata Harvesting. It complies to the specification of OAI Static Repository Gateways.

We need the following information to make your OAI Static Repository harvestable via our Gateway:

The network-location (URL) of your OAI Static Repository
The email address of the administrator of your OAI Static Repository

OAI Static Repositories can use our Gateway services if they meet the following technical conditions:

  • The baseURL element of Static Repository Files registered at this gateway must match:

    http://purl.oclc.org/NET/ugent/lib/srepod/<network address static repository file>


     E.g:

    <oai:baseURL>http://purl.oclc.org/NET/ugent/lib/srepod/dor2.ugent.be/gateway/vb1.xml</oai:baseURL>
    <oai:baseURL>http://purl.oclc.org/NET/ugent/lib/srepod/dor2.ugent.be/gateway/vbns.xml</oai:baseURL>
    <oai:baseURL>http://purl.oclc.org/NET/ugent/lib/srepod/somewhere.org/~jdoe/repo.xml</oai:baseURL>

    Examples Static Repository Files:

    http://dor2.ugent.be/gateway/vb1.xml
    http://dor2.ugent.be/gateway/vbns.xml

  • The OAI Static Repository must conform with the OAI Static Repository specification. Specifically:
    • The OAI Static Repository must validate against the XML Schema that defines the format for OAI Static Repositories.
    • The network-location of the OAI Static Repository must be an HTTP address, i.e. the network-location must start with 'http://'.
  • In addition, our Gateway imposes the following conditions:
    • The network-location of the OAI Static Repository must have a suffix '.xml' , i.e. the network-location must end on '.xml'
    • The network-location of the OAI Static Repository must only contain characters from the following set: [a-z] [A-Z] [0-9] '/' '+' '-' '%' '.' ':' '_' '~'
    • The size of the OAI Static Repository must not exceed 2 Mb.
